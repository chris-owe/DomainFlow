# 清流 PRD

## 一、产品概述

### 产品定位

- **产品名称**：清流
- **一句话定位**：混乱之中，流向清晰。把分散在各处的念头自动整理成有序的结构，让混乱变成一条清澈的流。
- **目标用户**：个人用户，优先解决自己"念头流失"的痛点，未来可扩展到有类似痛点的人群。

### 核心痛点

用户在个人事务管理上面临的四大困境：

1. **念头太多**：脑子里同时装着学习、工作、生活很多事，互相干扰
2. **工具太散**：备忘录、微信收藏、邮件、日历……每件事存在不同地方，找不到
3. **任务太大**：一个模糊的大目标不知道从哪里拆解，拖着不做
4. **没有节奏**：不知道今天该做什么，做完也不知道完成了多少

## 二、核心闭环

> 💡 **产品的核心逻辑**：收集 → 整理（结构化）→ 执行 → 回顾。其中「整理」是关键——不是简单展示，而是自动分类、设优先级、拆解任务，让混乱进来、清晰出去。四个环节首尾相连形成闭环。

| 环节 | 解决的问题 |
|------|-----------|
| **收集** | 任何场景快速记下念头，1秒入口，不让念头流失 |
| **整理** | 自动分类、设优先级、拆解任务，让混乱进来、清晰出去 |
| **执行** | 每日待办可视化，推进任务完成 |
| **回顾** | 定期复盘，知道自己完成了什么，减少拖延感 |

## 三、MVP 功能分级

### P0 — 第一版必须有的

| 功能 | 说明 |
|------|------|
| 便签式快速添加 | 打开 App/网页，一句话就能记下念头 |
| 今日待办视图 | 每天早上/前一天晚上看到今天要做什么 |
| 完成标记 | 点击即完成，有打勾的成就感 |
| 截止日期 + 提醒 | 设置提醒时间，到点推送通知 |

### P1 — 第二版加的

- 任务分类/标签：学习/生活/工作分开看
- 收集箱（Inbox）：临时存放区，随时清空
- 周回顾统计：这周完成了多少，拖延率是多少

### P2 — 后续迭代

- 数据分析、习惯追踪、日历同步等

## 四、产品形态

### Web + App 共用方案

Web 和 App 共用同一套数据底层，前端分开做 UI，保证两端的用户体验各自适配。

| 层次 | 推荐技术 | 说明 |
|------|---------|------|
| 前端 | React + Tailwind | 跨端体验好，社区资源丰富 |
| 数据服务 | Supabase | 免费、有数据库、实时同步，支持 Web + App 共用 |
| 发布 | Web 部署公网 + App 用 H5 壳 | App 端可嵌套 H5 页面降低成本 |

## 五、便签入口设计

### 设计原则

越轻越好：3秒内完成一次记录，打开成本趋近于零。

### 三种形态选择

| 形态 | 描述 | 适合场景 |
|------|------|---------|
| 悬浮按钮 + 弹窗 | 页面角落放一个「+」按钮，点击弹出简易输入框 | Web / App |
| 常驻侧边栏便签区 | 固定在屏幕边缘，随时展开收起 | 多任务办公场景 |
| 桌面小组件 | 手机/电脑桌面常驻小组件，一键展开 | 习惯用桌面入口的人 |

## 八、品牌设计

### 品牌定位

- **产品名**：清流
- **品牌口号**：混乱之中，流向清晰。
- **Tagline**：One clear flow from chaos to order.
- **品牌理念**：不是「一次做一件」，而是「在混乱中建立有序结构」——让混乱进来，让清晰出去。

### 视觉识别

- **色彩体系**：冷色清澈系，主色 #06B6D4（青色），辅助 #0EA5E9（天蓝），背景 #0F172A（深空黑），点缀 #38BDF8（亮青）。整体传达「清澈、通透、有序」的感觉。
- **设计语言**：几何极客风（Geometric Geek）——简洁线条、中等圆角、像素/几何图标风格。
- **Logo 方向**：像素几何风格，蓝色系，核心视觉元素「1」或「✓」，极简科技感。
