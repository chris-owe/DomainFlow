# 域流 v1 实施文档

> 基于《域流PRD_v2.md》的第一版（MVP）详细实施计划
> 目标：快速出可用的 Web 版本，自己先用起来
> 预估周期：2-3 周

---

## 一、v1 范围

### 功能清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 领域 CRUD | 创建/编辑/删除领域，默认四个领域 | P0 |
| 任务 CRUD | 创建/编辑/删除/完成任务 | P0 |
| 重复任务 | 按星期重复（如周一三五） | P0 |
| 笔记 CRUD | 创建/编辑/删除笔记（Markdown 编辑器） | P0 |
| 今日待办 | 聚合所有领域的今日任务 | P0 |
| 任务列表 | 按领域查看、筛选、排序 | P0 |
| 收集箱 | 快速捕获，7 天倒计时提醒 | P1 |
| 快速添加 | 全局悬浮按钮，快速记任务/笔记 | P1 |

### v1 不包含（留到 v2）

- 笔记-任务关联
- 标签系统
- 全局搜索
- 周回顾统计
- 日历视图

## 二、数据模型（Supabase/PostgreSQL）

### 表结构

#### domains（领域）

```sql
create table domains (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  description text default '',
  color text default '#06B6D4',
  icon text default '📁',
  sort_order int default 0,
  created_at timestamp default now(),
  updated_at timestamp default now()
);
```

默认种子数据：

```sql
insert into domains (name, description, color, icon, sort_order) values
  ('工作', '开发任务、项目管理、技术文档', '#0EA5E9', '📁', 1),
  ('考试', '学习计划、笔记、复习', '#8B5CF6', '📁', 2),
  ('技能', '技能拓展、教程笔记、练手项目', '#10B981', '📁', 3),
  ('内容创作', '公众号/视频号/抖音等选题、素材、排期', '#F59E0B', '📁', 4);
```

#### tasks（任务）

```sql
create table tasks (
  id uuid primary key default gen_random_uuid(),
  domain_id uuid references domains(id) on delete cascade,
  title text not null,
  description text default '',
  status text not null default 'pending' check (status in ('pending', 'in_progress', 'done')),
  priority text not null default 'P2' check (priority in ('P0', 'P1', 'P2', 'P3')),
  due_date date,
  is_recurring boolean default false,
  recurring_days int[] default null,  -- [1,3,5] = Mon/Wed/Fri
  sort_order int default 0,
  created_at timestamp default now(),
  updated_at timestamp default now()
);

-- 索引
create index tasks_domain_id_idx on tasks(domain_id);
create index tasks_status_idx on tasks(status);
create index tasks_due_date_idx on tasks(due_date);
```

#### notes（笔记）

```sql
create table notes (
  id uuid primary key default gen_random_uuid(),
  domain_id uuid references domains(id) on delete cascade,
  title text not null default '',
  content text default '',  -- markdown
  sort_order int default 0,
  created_at timestamp default now(),
  updated_at timestamp default now()
);

create index notes_domain_id_idx on notes(domain_id);
```

#### inbox_items（收集箱）

```sql
create table inbox_items (
  id uuid primary key default gen_random_uuid(),
  content text not null,                    -- 记下来的内容
  item_type text not null default 'task' check (item_type in ('task', 'note')),
  processed boolean default false,          -- 是否已归类
  deadline date not null,                   -- 默认为创建时间 + 7 天
  created_at timestamp default now()
);

create index inbox_items_deadline_idx on inbox_items(deadline);
```

## 五、详细功能设计

### 5.1 今日待办（默认首页）

**逻辑：**
- 展示所有领域中状态为 `pending` 且满足以下条件的任务：
  - `due_date = today` 的所有任务
  - 无截止日期的 P0 / P1 任务
  - 今天的重复任务实例
- 按优先级排序：P0 → P1 → P2 → P3
- 勾选后任务状态变为 `done`，从列表消失（可查看"已完成"）
- 空状态：显示"今天没有待办事项，享受平静的一天"

**交互细节：**
- 点击任务标题 → 弹出编辑弹窗
- 点击勾选框 → 直接完成（带一个简短的勾选动画）
- 过期任务（due_date < today）用红色标识

### 5.2 任务管理

**创建任务弹窗：**

```
┌─────────────────────────┐
│  新建任务               │
│                         │
│  任务标题 *             │
│  ┌───────────────────┐  │
│  │                   │  │
│  └───────────────────┘  │
│                         │
│  所属领域      [工作 ▼] │
│  优先级       [P2 ▼]   │
│  截止日期     [选择日期] │
│                         │
│  □ 重复任务             │
│    重复星期: □一 □二 □三│
│              □四 □五 □六│
│              □日        │
│                         │
│  描述（选填）            │
│  ┌───────────────────┐  │
│  │                   │  │
│  │                   │  │
│  └───────────────────┘  │
│                         │
│         [取消] [创建]   │
└─────────────────────────┘
```

**重复任务的实现逻辑：**
1. 创建时勾选"重复任务"并选择星期
2. 当某天的实例被标记为 `done` 后，自动计算下一次执行日期
3. 计算规则：从今天之后，找 `recurring_days` 中最近的一个未来日期
4. 例如周五完成了周一三五的任务，自动生成下周一的待办
5. 今日待办中按日期匹配展示

### 5.3 笔记管理

**笔记编辑器：**
- MVP 使用基础 Markdown 编辑（纯文本 + 预览模式切换）
- 标题在上方，内容在下方
- 保存后自动更新时间

**笔记列表：**
- 显示标题 + 前两行内容摘要 + 更新时间
- 按更新时间倒序
- 点击进入编辑

### 5.4 领域管理

**侧边栏领域操作：**
- 点击领域 → 进入该领域视图
- 鼠标悬停显示编辑/删除按钮（或右键菜单）
- "＋添加领域"在领域列表底部
- 删除领域时二次确认

**创建/编辑领域弹窗：**

```
┌──────────────────────┐
│  编辑领域             │
│                      │
│  名称 *              │
│  ┌────────────────┐  │
│  │ 工作           │  │
│  └────────────────┘  │
│                      │
│  描述                │
│  ┌────────────────┐  │
│  │ 开发任务...     │  │
│  └────────────────┘  │
│                      │
│  颜色    ○  ○  ○  ○  │
│          蓝色 紫色 绿色 │
│                      │
│        [取消] [保存]  │
└──────────────────────┘
```

### 5.5 收集箱

**入口：** 侧边栏 "📥 收集箱"
**空状态：** 显示"收集箱已清空，没有待归类的信息"
**有内容时：**
- 每条内容显示：文本内容 + 类型标签（笔记/任务）+ 剩余天数倒计时
- 剩余 3 天以内：黄色提示
- 超期（剩余 < 0 天）：红色高亮
- 点击"归类"→ 弹出选择领域弹窗，选择后从收集箱移至对应领域
- 点击"删除"→ 直接丢弃

**倒计时规则：**
- 创建时 `deadline = 创建当天 + 7 天`
- 展示格式："剩余 5 天" / "今天截止" / "已超期 2 天"

### 5.6 所有内容

**不选领域时，跨领域展示：**
- 页签：全部 / 笔记 / 任务
- "全部" = 笔记 + 任务按时间混排
- 每条内容显示所属领域（带领域颜色标识）
- 按更新时间倒序

每个列表视图都有对应的空状态：

| 视图 | 空状态文案 |
|------|-----------|

| 所有内容 | 还没有内容，从创建第一个任务或笔记开始 |

## 八、验收标准

部署后自己验证以下流程：

### 核心流程
- [ ] 打开页面，看到默认四个领域
- [ ] 在"工作"领域创建一个任务，设今天截止
- [ ] 在"技能"领域写一条笔记
- [ ] 打开今日待办，能看到刚才创建的任务
- [ ] 勾选完成任务，任务从待办消失
- [ ] 新建一个重复任务（周一三五），勾选完成，确认自动生成下一次

### 收集箱流程
- [ ] 通过收集箱快速记一条内容
- [ ] 看到倒计时显示剩余 X 天
- [ ] 归类到某个领域
- [ ] 确认内容出现在对应的领域列表中

### 边界情况
- [ ] 删除领域时，该领域下的任务和笔记如何处理？（警告提示，确认后全部删除）
- [ ] 截止日期为今天的任务在今日待办中展示
- [ ] 过期的任务红色标识
- [ ] 收集箱超期内容红色高亮
- [ ] 编辑任务/笔记后内容正确保存
